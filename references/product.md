# Product Design and MVP

Use this file for product strategy, MVP, feature prioritization, user-problem analysis, product scope, and architecture questions.

## Core orientation

Do not assume the requested feature is the real need.

Start from:

- user job
- context of use
- current failure mode
- friction in the workflow
- minimum capability needed to solve the job
- fastest route to user value and learning

## Reframe common questions

### Should we build this feature?

Reframe as:

- What user outcome is currently blocked?
- Is this feature the smallest intervention that unblocks it?
- Is this feature solving the real problem or compensating for a deeper design weakness?

### What should the MVP include?

Reframe as:

- What is the smallest product that creates a meaningful outcome?
- What is the minimum evidence needed to validate demand?
- What is the minimum implementation needed to generate that evidence?

Do not confuse MVP with a smaller version of the full roadmap.

### How should we improve the product?

Reframe as:

- Which user outcome matters most?
- Where does the current experience fail?
- Which friction matters most for activation, retention, or monetization?
- What can be removed while preserving value?

## Distinguish layers

Separate:

- user outcome
- workflow
- interface
- feature
- implementation detail

Many teams inherit interface patterns and mistake them for requirements.

## Useful output structure

- User's real job
- Assumptions in the request
- Basic product facts
- Hard constraints
- Soft constraints
- Minimum product shape
- What to ship now
- What to defer
- How to validate
